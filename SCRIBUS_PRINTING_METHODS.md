# Scribus SLA Printing Methods

This document explains the different methods available for opening and printing Scribus SLA files, including both command-line and library-based approaches.

## Available Print Methods

### 1. Library Method (Recommended) - `--print-method library`

**Pure Python libraries approach - No Scribus command line required**

- **Advantages:**
  - No dependency on Scribus command line
  - Faster execution
  - More reliable on Windows
  - Better error handling
  - Self-contained solution

- **How it works:**
  1. Parses SLA XML files using standard Python XML libraries
  2. Extracts layout information (images, text, positions)
  3. Converts to PDF using ReportLab library
  4. Prints PDF using Windows system commands

- **Requirements:**
  ```bash
  pip install reportlab pillow pywin32
  ```

- **Usage:**
  ```bash
  python create_scribus_files.py deck.xml --print-method library --print
  ```

### 2. Command Line Methods

#### Scribus Direct - `--print-method scribus`
- Uses Scribus command line with Python scripts
- Requires Scribus installation
- May have driver dialog issues on Windows

#### PDF Export Method - `--print-method pdf`
- Exports SLA to PDF via Scribus command line
- Then prints PDF using system commands
- More reliable than direct Scribus printing
- Requires Scribus installation

### 3. Manual Method - `--print-method manual`
- Opens SLA files in Scribus GUI for manual printing
- User controls each print operation
- Most reliable but not automated

## Troubleshooting

### Library Method Issues
- **Missing libraries:** Run `pip install reportlab pillow pywin32`
- **Image not found:** Check relative paths in SLA files
- **Print failures:** Verify printer name and status

### Command Line Issues
- **Scribus not found:** Install Scribus or set `SCRIBUS_CMD` environment variable
- **Timeout errors:** Increase timeout values or use library method
- **GUI dialogs:** Use `--no-headless` flag for debugging
